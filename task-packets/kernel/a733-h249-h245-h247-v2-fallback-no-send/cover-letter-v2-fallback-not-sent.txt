Subject: [RFC/RFT v2 option 0/4] clk: sunxi-ng: keep Cubie A7S SDMMC0 path live

This is a v2 option for the A733 CCU/NSI series. It keeps the same
Cubie A7S tested critical-clock set as the earlier RFC/RFT, but expresses
the NSI update-bit fix in common sunxi-ng registration code instead of as
an A733-specific probe-time write.

The common update-bit patch is intentionally broader than the A733-specific
form: it commits boot-programmed state once for every clock carrying
CCU_FEATURE_UPDATE_BIT in a registered CCU descriptor. That may be preferable
if reviewers want the feature flag to own this semantic, but it should not
replace the narrow A733 form unless maintainers ask for common handling.

This common helper is scoped to the registration-time handoff from firmware
to Linux. It does not claim to audit every later set_rate() path for
update-bit clocks; if reviewers want broader framework cleanup for future
rate changes, that should be handled as a separate follow-up.
