## Current recommendation

Keep H153 as the default upstream-shape candidate until hardware proof says otherwise.

Reason:

- H153 preserves the known-good diagnostic bundle that reached `mmcblk0`, discovered `p1 p2 p3`, mounted root read-only, and started `/bin/sh`.
- H154 is cleaner if it passes, but it is not yet hardware-proven.
- Removing `mbus-msi-lite0` before proof would narrow the patch but weaken the evidence chain.
