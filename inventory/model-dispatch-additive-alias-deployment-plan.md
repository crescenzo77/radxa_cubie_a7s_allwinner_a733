## Purpose

Plan a future additive `model-dispatch` alias deployment from the reviewed
source repo at `/srv/projects/model-dispatch`.

`/srv/model-dispatch`, edit `/srv/projects/model-dispatch`, restart services,
deploy aliases, change Open WebUI config, change OpenCode, change Continue.dev,

For `free-cloud`, the later source change must make
`free-cloud -> openrouter-free/openrouter/auto-free-router` an exact alias.
Because current `tests/check_config.py` requires route targets to exist in
`models`, and current `app.py` handles OpenRouter-free IDs outside `models`,
the future implementation should either:

- extend the config validator and route resolver to allow routes that target
  generated `FREE_BY_DISPATCH_ID` entries, then add:

```json
{
  "free-cloud": [
    "openrouter-free/openrouter/auto-free-router"
  ]
}
```

- or add a dedicated reviewed alias map in `app.py` for OpenRouter-free aliases.

The first option best matches this slice because it keeps alias behavior in the
route registry. It still requires review before deployment because it changes
application behavior, not only `config.json`.

## Files Eligible for Deployment

For an alias-only deployment, only these reviewed source files are eligible:

- `app.py`, only if needed for `free-cloud` alias support or validator-aligned
  route behavior.
- `config.json`.
- `.gitignore`, only if changed in the reviewed source commit.
- `.cgcignore`, only if changed in the reviewed source commit.

Docs and tests remain source-repo artifacts by default. They should not be
copied to `/srv/model-dispatch` unless the operator explicitly approves
deploying docs/tests to the live runtime path.

## Backup Path

Before any future deployment, create a timestamped backup under:

```text
/srv/model-dispatch/backups/<timestamp>/
```

Back up these live files if present:

- `/srv/model-dispatch/app.py`
- `/srv/model-dispatch/config.json`
- `/srv/model-dispatch/.gitignore`
- `/srv/model-dispatch/.cgcignore`

Do not include logs, caches, secrets, env files, virtualenvs, databases, request
logs, generated runtime files, or `.git/`.

## Rollback Plan

Rollback for the first alias deployment should remove only the additive alias
behavior and preserve all existing IDs.

Future rollback steps:

1. Restore `config.json` from
   `/srv/model-dispatch/backups/<timestamp>/config.json`.
2. Restore `app.py` from `/srv/model-dispatch/backups/<timestamp>/app.py` if
   the deployment changed `app.py`.
3. Restore `.gitignore` and `.cgcignore` from the same backup if they were
   changed.
4. Validate Python syntax and JSON syntax.
5. Restart or reload `model-dispatch.service` only after explicit operator
   approval.
6. Confirm `/health`, `/v1/models`, and a preserved ID such as `auto-local`
   work again.

OpenCode direct AMD routing remains the OpenCode rollback path. Continue.dev's
existing documented route remains its rollback path. LiteLLM remains
rollback/history until a separate approved slice changes that posture.
